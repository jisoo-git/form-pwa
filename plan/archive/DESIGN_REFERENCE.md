# 디자인 레퍼런스 — awesome-design-md 요약

> **요약 (3줄):**
> 1. **무엇을 쓰는 파일?** Linear·Stripe·Vercel·Notion 디자인 MD를 읽고 인코딩플러스에 적용할 수 있는 패턴을 정리한 외부 레퍼런스 요약.
> 2. **현재 상태?** 참고 자료 (설계 결정 근거) — 섹션 구분선(`#e3e8ee`), 배경 교차(`#f6f9fc`), 카드 패딩 등 이미 적용된 것도 포함.
> 3. **언제 참고?** 새 UI 패턴이나 레이아웃 변경 시 "어떤 회사에서 어떤 방식을 썼나" 확인할 때. DESIGN.md와 함께 쓰면 좋음.

## 1. Linear

**특징:** 다크 테마, 라벤더-블루 포인트, 제품 스크린샷 중심

| 항목 | 값 |
|------|----|
| 포인트 색 | `#5e6ad2` (라벤더-블루) |
| 기본 단위 | 4px |
| 상단 네비 | 56px sticky |
| 카드 패딩 | 24px 기본 / 32px 테스티모니얼 |
| 버튼 패딩 | `8px 14px` |
| 반경 | 16px (카드), focus ring 2px |
| 섀도우 | 거의 없음 — surface 색상 계층으로 대체 |
| 타이포 | 디스플레이 80px → 바디 16px, 무게 600→400 |
| 자간 | 디스플레이 -3.0px → 바디 0 |

**우리한테 적용 가능:**
- 4px 기본 단위 시스템 (spacing 통일)
- 56px sticky TopNav ✓ 이미 사용 중
- 섀도우 없이 border+bg색으로 계층 표현

## 2. Stripe

**특징:** 화이트 캔버스, 인디고 포인트, 그라디언트 메시 히어로

| 항목 | 값 |
|------|----|
| 섹션 회색 배경 | `#f6f9fc` (파란기 있는 그레이) |
| 구분선/테두리 | `#e3e8ee` |
| 본문 텍스트 | `#0d253d` |
| 보조 텍스트 | `#273951` |
| 흐린 텍스트 | `#64748d` |
| 포인트 | `#533afd` (인디고) |
| 카드 반경 | 12px (`lg`) |
| 버튼 | pill 형태 (`9999px`) |
| 버튼 패딩 | `8px 16px` |
| 카드 섀도우 | `rgba(0,55,112,0.08) 0 1px 3px` (Level 1) |
| 카드 패딩 | 32px |
| 입력창 반경 | 6px |
| 입력창 포커스 | 인디고 border |
| 태그/뱃지 | pill, 연한 배경 + 진한 텍스트 |

**우리한테 적용 가능:**
- 섹션 회색 `#f4f4f6` → `#f6f9fc` 교체
- 구분선 `#e4e4e7` → `#e3e8ee` 교체
- 카드에 Level 1 섀도우 추가
- 카테고리 뱃지 pill 스타일

## 3. Vercel

**특징:** 블랙/화이트 미니멀, 타이포그래피로 계층 표현

| 항목 | 값 |
|------|----|
| 포인트 | 검정 `#171717` 단색 |
| 캔버스 | `#fafafa` 소프트 화이트 |
| 기본 단위 | 4px |
| 섹션 간격 | 64–96px |
| 마케팅 버튼 | pill (`100px`) |
| 네비 버튼 | `6px` 반경 |
| 타이포 자간 | -2.4px(디스플레이) → -0.6px |

**우리한테 적용 가능:**
- 섹션 수직 패딩 64–96px (현재 36px — 좀 더 여유롭게)
- 디스플레이 텍스트 자간 강화

## 4. Notion

**특징:** 다크 네이비 히어로, 퍼플 CTA, 파스텔 카드

| 항목 | 값 |
|------|----|
| CTA 색 | `#5645d4` (퍼플) |
| 카드 반경 | 12px (모든 카드 통일) |
| 버튼 반경 | 8px (직사각형, pill 아님) |
| 버튼 패딩 | `10px 18px` |
| 카드 패딩 | 32px |
| 입력창 높이 | 44px |
| 입력창 포커스 | 2px border |
| 뱃지 반경 | 6px (태그), 9999px (pill 탭만) |
| 섀도우 Level 2 | `rgba(15,15,15,0.08) 0 8px 24px` |
| 섹션 간격 | 96px |

**우리한테 적용 가능:**
- 카드 반경 12px으로 통일 (현재 14px–18px 혼재)
- 버튼 반경 8px–10px (현재 10–12px, OK)
- 입력창 44px 높이 표준
- 파스텔 배경 카드 (WHY 섹션 등에 적용 가능)
- 비교 테이블 패턴 (Courses 페이지에 이미 있음)
- FAQ 아코디언 패턴

## 인코딩플러스 적용 우선순위

### 즉시 적용 가능 (색상/수치)
| 항목 | 현재 | 변경 | 출처 |
|------|------|------|------|
| 섹션 회색 bg | `#f4f4f6` | `#f6f9fc` | Stripe |
| 구분선 | `#e4e4e7` | `#e3e8ee` | Stripe |
| 흐린 텍스트 | `#9b9ba5` | `#8c959f` | Stripe/Notion |
| 카드 섀도우 | 없음 | `rgba(0,55,112,0.06) 0 1px 4px` | Stripe Level 1 |
| 카드 반경 | 14–18px 혼재 | 12px 통일 | Notion |
| 카테고리 뱃지 | `borderRadius: 6px` | pill (`9999px`) | Stripe/Notion |

### 구조적 개선 (시간 필요)

- 타이포그래피 자간 강화 (`letterSpacing: '-0.03em'` 이미 일부 적용)
- FAQ 아코디언 추가 (Courses 페이지)
