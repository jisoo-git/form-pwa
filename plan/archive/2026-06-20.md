## 현재 동작 상태

### ✅ 로컬(localhost:5173)에서 동작 확인됨
- 관리자 로그인 (`/admin`) — 비밀번호 `coding1234`
- 대시보드 (`/admin/dashboard`) — 폼 목록, 활성/비활성 토글, 삭제
- 폼 빌더 (`/admin/builder`) — 섹션·질문 추가/편집/저장, 드래그 정렬
- 폼 저장 — Firestore에 정상 저장 (undefined 필드 제거 버그 수정 완료)
- 홈 연결 설정 — 단기특강/일반전형 버튼에 폼 ID 직접 지정, Firestore `settings/landing`에 저장
- 응답 보기 (`/admin/responses/:id`) — 응답 목록, 클릭 시 답변 상세, 퀴즈 점수
- 공개 폼 페이지 (`/form/:id`) — 섹션별 진행, 이전 버튼, 제출 후 홈 이동
- 랜딩 페이지 (`/`) — 단기특강(blue-800)/일반전형(violet-800) 색상 구분, 시간표 카드

### ⚠️ Vercel 배포 환경에서 미확인/미동작
- **관리자 로그인 안 됨** — Vercel에 `VITE_ADMIN_PASSWORD` 환경변수 미설정
- **Firestore 연결 미확인** — 나머지 Firebase 환경변수 6개도 미설정
- **홈 신청 버튼** — 환경변수 없으면 Firestore 못 읽으니 "신청 준비 중" 상태

### 🔲 구현됐지만 실제 테스트 안 한 기능
- PDF 업로드 (Storage rules 배포 안 됨 — `firebase deploy --only storage` 미실행)
- OMR 질문 (빌더 UI는 있으나 실제 응답 제출 테스트 안 함)
- 섹션 분기 + ⛔ 진행 불가 (코드 완성, 테스트 미완)
- 퀴즈 자동 채점 (코드 완성, 테스트 미완)

## 오늘 구현한 기능 목록

### Phase 5 — 공개 폼 응답 페이지 `/form/:id`
- 폼 로드 (비활성 폼이면 "찾을 수 없음" 화면)
- 섹션별 순차 진행 + 프로그레스 바
- 이전 섹션으로 돌아가기 (히스토리 스택)
- 필수 항목 미입력 시 빨간 테두리 + 오류 문구
- 모든 질문 유형 렌더링:
  - 단답형, 장문형, 숫자, 날짜
  - 객관식(radio), 다중선택(checkbox), 드롭다운
  - O/X, OMR(원문자 ①②③...), PDF 열기
- 섹션 분기 (radio/dropdown/OX 답변 기반)
- ⛔ 진행 불가 분기 — 해당 답변 선택 시 다음 버튼 비활성화
- 퀴즈 타입: 제출 시 자동 채점 → 점수/정답률 표시
- 제출 완료 1초 후 홈(`/`)으로 자동 이동
- 고정 이름 입력 제거 (관리자가 폼에 질문으로 직접 추가)

### Phase 6 — 관리자 응답 목록 `/admin/responses/:formId`
- 응답 목록: 응답자 이름, 제출 시간 (최신순)
- 클릭 시 질문별 답변 펼쳐보기
- 퀴즈 타입: 개별 점수 + 전체 평균 점수 표시

### 폼 빌더 개선
- 선택지 삭제 X 버튼 항상 표시 (기존: 호버해야 보임)
- OX 질문에 분기 설정 추가
- 분기 드롭다운에 ⛔ 진행 불가 옵션 추가

### 대시보드 개선
- 링크 복사 버튼 — `/form/:id` URL 클립보드 복사
- 🔗 홈 연결 설정 모달 — 각 코스 버튼에 폼 직접 지정
- 저장/토글 실패 시 alert로 에러 메시지 표시

### 버그 수정
| 증상 | 원인 | 해결 |
|------|------|------|
| 폼 저장 실패 | 질문 optional 필드 `undefined` 값 | `JSON.parse(JSON.stringify())` 로 제거 |
| `/admin` 빈 화면 | `navigate()` 렌더 중 호출 | `useEffect`로 이동 |
| 응답 목록 안 뜸 | `where + orderBy` 복합 쿼리 인덱스 없음 | `orderBy` 제거 후 클라이언트 정렬 |
| 홈 연결 설정 저장 안 됨 | `settings` 컬렉션 Firestore 규칙 없음 | 규칙 추가 후 배포 |

## 내일 해야 할 것

### 🔴 최우선 (배포 환경 정상화)
1. **Vercel 환경변수 7개 입력**
   - Vercel → Settings → Environment Variables
   ```
   VITE_FIREBASE_API_KEY
   VITE_FIREBASE_AUTH_DOMAIN
   VITE_FIREBASE_PROJECT_ID
   VITE_FIREBASE_STORAGE_BUCKET
   VITE_FIREBASE_MESSAGING_SENDER_ID
   VITE_FIREBASE_APP_ID
   VITE_ADMIN_PASSWORD = coding1234
   ```
   - 입력 후 Deployments → Redeploy 필수

2. **배포 환경 전체 동작 확인**
   - 관리자 로그인 → 폼 목록 뜨는지
   - 폼 만들고 활성화 → 홈 신청 버튼 연결 → 학생 입장에서 폼 제출 → 응답 확인
